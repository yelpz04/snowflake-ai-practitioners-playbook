# Day 11: Cortex Code Skill for DCM Projects

## Goal
Create a declarative pipeline project and use Cortex Code's DCM skill to validate and deploy.

## References
- [DCM blog](https://www.snowflake.com/en/blog/declarative-snowflake-pipelines-dcm-projects-cortex-code/)

## What to Do
1. Review `dcm/manifest.yml` — the declarative project definition
2. Review `dcm/define_tables.sql` and `dcm/define_tasks.sql`
3. In Cortex Code, run: "Analyze my DCM project for issues"
4. Run: "Plan the deployment — what will change?"
5. Deploy to a dev environment

## Key Concepts
- DCM = Declarative Configuration Management
- `manifest.yml` defines the project structure
- Cortex Code DCM skill: analyze → plan → deploy
- Infrastructure-as-code for Snowflake objects
